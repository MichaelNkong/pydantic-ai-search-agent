# AI Agents with Pydantic

This is a simple Streamlit-based application demonstrating an AI search agent built using the `pydantic-ai` library. The project lets users enter search queries and retrieves recent web results using the `ddgs` library.

## Features

- Streamlit UI for entering queries
- Pydantic-based agent configuration
- Tool for internet search using DuckDuckGo (via `ddgs`)
- Example of handling network errors and normalizing tool output

## Requirements

Python 3.10+ (a virtual environment is recommended)

Dependencies are listed in `requirements.txt`. Install them with:

```bash
python -m pip install -r requirements.txt
```

## Setup

1. Clone the repository.
2. Create and activate a virtual environment:
   ```bash
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1  # PowerShell
   # or
   source .venv/bin/activate      # macOS / Linux
   ```
3. Install requirements (see above).
4. Create a `.env` file if you want to set environment variables for the OpenAI model key (e.g., `OPENAI_API_KEY`).

## Running the app

Start the Streamlit application:

```bash
streamlit run main.py
```

Then open the URL printed by Streamlit (usually http://localhost:8501) in your browser.

Enter a query and click **Search** to see results.

## Project Structure

```
main.py               # Streamlit front-end
agents/
  search_agent.py     # Agent configuration
tools/
  search_internet.py  # Web search tool (DDGS)
  schema.py           # Pydantic input/output models

```

## Troubleshooting

- **DDGSException / network errors**: The search tool now catches exceptions and returns an empty list. Ensure your machine has internet access and no firewall is blocking requests to DuckDuckGo or Yandex.
- **Async errors in Streamlit**: The app normalizes asynchronous output, but avoid using `asyncio.run()` inside callbacks if Streamlit reports event loop errors.

## Challenges / Next Steps

These are suggested challenges and improvements you can try to extend this project:

- **Robust result parsing**: Improve the search tool to return strongly-typed `SearchResult` objects (use `SearchOutput`) and ensure the agent always returns consistent JSON.
- **Retry & backoff**: Add retry logic with exponential backoff for transient DDGS/network failures and surface user-friendly error messages in the UI.
- **Multiple search backends**: Add support for alternative search backends (Google Custom Search, Bing, SerpAPI) with a pluggable backend interface and feature flags.
- **Caching & rate-limiting**: Cache recent queries and implement rate limiting to reduce repeated network calls and avoid provider throttling.
- **Pagination & result metadata**: Support paginated results, include source metadata (publish date, domain), and allow users to request more pages.
- **Testing & CI**: Add unit tests for `tools/search_internet.py` (mock DDGS), integration tests for the agent, and a simple GitHub Actions workflow to run tests.
- **Secure secrets management**: Move API keys out of `.env` into a secrets manager or GitHub Actions secrets; validate missing/invalid keys at startup.
- **Docker + deployment**: Add a `Dockerfile` and `docker-compose.yml` so the app can be run in containers and deployed to a cloud service.
- **UI improvements**: Enhance the Streamlit UI — add loading states, result filtering, grouping by source, and a compact/list view toggle.
- **Usage analytics**: Add optional telemetry (with user consent) to track query volume and failures to identify reliability issues.
